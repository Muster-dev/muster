# Payload Signing

> Ed25519/RSA payload signing and verification for fleet integrity.

Payload signing ensures that agent scripts, config files, and hook payloads have not been tampered with. The system prefers Ed25519 when available, falling back to RSA-2048 on older OpenSSL versions.

## How It Works

1. A signing keypair is generated on the fleet operator's machine
2. Files are signed with the private key, producing a base64 signature
3. The public key and signatures are distributed to remote agents
4. Agents verify signatures before executing any signed file

## Algorithm Selection

The system auto-detects the best available algorithm:

| Priority | Algorithm | Condition |
|----------|-----------|-----------|
| 1 | Ed25519 | `openssl genpkey -algorithm ed25519` succeeds |
| 2 | RSA-2048 | Fallback for older OpenSSL (< 1.1.1) |

Once selected, the algorithm is stored in `~/.muster/fleet/keys/keytype` and reused for all subsequent operations.

## Key Storage

| File | Permissions | Description |
|------|-------------|-------------|
| `~/.muster/fleet/keys/payload.key.pem` | `600` | Private signing key |
| `~/.muster/fleet/keys/payload.pub.pem` | `644` | Public verification key |
| `~/.muster/fleet/keys/keytype` | `644` | Algorithm identifier (`ed25519` or `rsa`) |
| `~/.muster/fleet/authorized_keys/` | `700` | Directory of trusted keys (target-side) |

## Signing and Verification

**Sign a file:**

```bash
# Internal function — called by install-agent and app manifest signing
payload_sign <file>
# Output: single line of base64 (the signature)
```

Uses `openssl pkeyutl -sign` with the private key. Output is base64-encoded on a single line.

**Verify a file:**

```bash
# Internal function — called by agent on startup
payload_verify <file> <sig_b64> <pubkey_file>
# Returns: 0 = valid, 1 = invalid
```

Decodes the base64 signature to a temp file, then runs `openssl pkeyutl -verify`.

## Fingerprints

Public keys are identified by a short SHA-256 fingerprint:

```
mst:a1b2c3d4e5f67890
```

Format: `mst:` prefix + first 16 hex characters of `SHA256(DER-encoded public key)`.

Generated by `payload_fingerprint()`.

## Authorized Keys (Trust Management)

Remote machines maintain a list of trusted public keys for verifying incoming payloads.

### Trust a Key

```bash
muster fleet trust-key
```

Adds a public key to the authorized keys store. Keys are stored as JSON files in `~/.muster/fleet/authorized_keys/<fingerprint>.json`:

```json
{
  "keys": [
    {
      "label": "operator-laptop",
      "pubkey": "-----BEGIN PUBLIC KEY-----\n...",
      "added": "2026-03-04T12:00:00Z"
    }
  ]
}
```

### List Trusted Keys

```bash
muster fleet list-keys
```

Shows all trusted keys with their label, fingerprint, and timestamp.

### Revoke a Key

```bash
muster fleet revoke-key <label>
```

Removes a key by label from the authorized keys store.

## Functions

All functions are defined in `lib/core/payload_sign.sh`.

| Function | Description |
|----------|-------------|
| `payload_sign <file>` | Sign a file, output base64 signature to stdout |
| `payload_verify <file> <sig_b64> <pubkey>` | Verify a file against a signature (returns 0/1) |
| `payload_fingerprint [pubkey]` | SHA-256 fingerprint of a public key (`mst:...` format) |
| `payload_trust_key <pubkey_file> <label> <fp>` | Add a key to the authorized keys store |
| `payload_list_keys <fingerprint>` | List all trusted keys for a fingerprint |
| `payload_revoke_key <label> <fingerprint>` | Remove a trusted key by label |
| `payload_read_pubkey` | Print the local public key PEM content |

## Used By

- **`muster fleet install-agent`** -- Signs the agent script and config, pushes signature + public key to the remote
- **Agent startup** -- Verifies agent script and config signatures before running
- **`muster fleet keygen`** -- Generates the signing keypair
- **App integrity** (`lib/core/app_verify.sh`) -- Signs the file manifest

## Dependencies

- `openssl` (standard on Linux and macOS)
- No external libraries or tools required
